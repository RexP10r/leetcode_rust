**Tags:** `#medium` `#concurrency` 

## Problem
Five silent philosophers sit at a round table with bowls of spaghetti. Forks are placed between each pair of adjacent philosophers.

Each philosopher must alternately think and eat. However, a philosopher can only eat when they have both their left and right forks. Each fork can be held by only one philosopher at a time. After finishing eating, a philosopher must put down both forks so they become available to others. A philosopher can pick up forks as they become available, but cannot start eating before acquiring both.

Design a concurrent algorithm (a discipline of behaviour) such that **no philosopher will starve** — i.e., each can forever continue to alternate between eating and thinking, assuming no philosopher knows when others may want to eat or think.

Implement the function:
```java
void wantsToEat(int philosopher, 
                Runnable pickLeftFork, 
                Runnable pickRightFork, 
                Runnable eat, 
                Runnable putLeftFork, 
                Runnable putRightFork)
```

Where:
- `philosopher` is the id of the philosopher who wants to eat (0 to 4, clockwise).
- `pickLeftFork` / `pickRightFork` are functions to pick the corresponding fork.
- `eat` is a function to let the philosopher eat once both forks are held.
- `putLeftFork` / `putRightFork` are functions to release the corresponding forks.

Five threads, each representing a philosopher, will simultaneously use one instance of your class. The function may be called for the same philosopher more than once, even before the last call ends.

## Examples
**Input:** `n = 1`  
**Output:** `[[3,2,1],[3,1,1],[3,0,3],[3,1,2],[3,2,2],[4,2,1],[4,1,1],[2,2,1],[2,1,1],[1,2,1],[2,0,3],[2,1,2],[2,2,2],[4,0,3],[4,1,2],[4,2,2],[1,1,1],[1,0,3],[1,1,2],[1,2,2],[0,1,1],[0,2,1],[0,0,3],[0,1,2],[0,2,2]]`  

*Explanation:*  
- `n` is the number of times each philosopher will call the function.  
- The output array describes calls to fork/eat functions in format: `output[i] = [a, b, c]`  
  - `a`: philosopher id (0–4)  
  - `b`: fork identifier `{1: left, 2: right}`  
  - `c`: operation `{1: pick, 2: put, 3: eat}`  

## Constraints
- `1 <= n <= 60`
- Philosophers are numbered `0` to `4` in clockwise order.
- The solution must prevent deadlock and starvation.
